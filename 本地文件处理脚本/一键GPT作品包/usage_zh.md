# 一键作品包使用说明

这个工具用于把“一篇笔记”的 GPT 文案和图片整理成一个作品包。

## 放在哪里

把工具安装到你平时下载图片的那个文件夹里。

文件夹里会有：

- `一键生成作品包.vbs`：平时只需要双击这个。
- `make_work_package.ps1`：后台执行脚本，通常会隐藏。
- `workpkg_config.json`：配置文件，可以改成品库名字和提示语。

## 怎么用

1. 先把这一篇笔记的图片下载到工具所在的这一层文件夹。
2. 再复制 GPT 生成好的文案。
3. 双击 `一键生成作品包.vbs`。
4. 工具会在成品库里生成一个新作品包。
5. 如果成品库一级散着的作品包凑满 14 个，工具会自动整理成下一个 `作品集_001 / 作品集_002 ...`。

## 生成结果

默认会生成：

```text
成品库\日期时间_文案第一行
```

作品包里面会有：

- `文案_日期时间.txt`
- `标题_日期时间_01.jpg / 标题_日期时间_02.png / 标题_日期时间_03.webp ...`

不会再创建空白分隔图。作品包里只放文案和你下载的作品图。

图片会按修改时间排序后再重命名，文件名带文案第一行标题和日期时间。工具也会给图片设置连续时间，尽量让手机按名称排序、按时间排序都更好分辨。

只处理工具所在这一层的图片，不会进入子文件夹。

## 作品集自动整理

默认规则：

- 只看成品库这一层的作品包文件夹。
- 已经叫 `作品集_001`、`作品集_002` 这种的文件夹会跳过。
- 每 14 个作品包自动收进一个新作品集。
- 每生成一个新作品集，会顺带生成同名 ZIP 压缩包，比如 `作品集_005.zip`。
- 不满 14 个就留在原地，等下次继续凑。
- 记录会保存在 `_portfolio_move_logs`。
- 如果触发整理和压缩，会按节点连续提示：`已创建作品包`、`已整理作品集`、`已生成ZIP压缩包`。

这个整理已经融入 `一键生成作品包.vbs`，平时不用再额外点分组脚本。旧的手动分组脚本可以作为补救工具保留。

## 重要规则

- 一条文案对应一篇笔记。
- 没复制文案时，会在屏幕中间显示白绿色圆角小提示“请先复制文案”，不会移动图片。
- 已经复制文案、但这一层没有作品图时，会提示“请先下载作品图”，不会生成空作品包，也不会清空剪贴板。
- 这篇文案已经做过作品包时，会提示“该作品已创建过，已清理本次重复下载”，并清理本次同层重复下载的图片。
- 就算旧作品包已经被收进 `作品集_001` 里面，也会继续参与查重。
- 成功后会清空剪贴板，减少误用上一条文案的概率。

## 修改成品库名字

打开 `workpkg_config.json`，修改：

```json
{
  "library_name": "成品库"
}
```

比如可以改成：

```json
{
  "library_name": "团建成品库"
}
```
