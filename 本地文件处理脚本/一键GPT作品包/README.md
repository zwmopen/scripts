# 一键 GPT 作品包

这个工具用于把“一篇笔记”的 GPT 文案和同层图片整理成一个作品包。

## 使用流程

1. 先把这一篇笔记的图片下载到脚本所在文件夹。
2. 再复制 GPT 生成好的文案。
3. 双击 `一键生成作品包.vbs`。
4. 工具会自动生成 TXT、重命名图片、移动到成品库。

## 当前规则

- 只处理脚本所在文件夹这一层的图片，不进入子文件夹。
- 没复制文案时，只提示“请先复制文案”，不整理、不移动。
- 有文案但没图片时，只提示“请先下载作品图”，不生成空包。
- 已经创建过的文案会被查重，重复下载的图片会自动清理。
- 图片会按 `标题_日期时间_01`、`标题_日期时间_02` 这种方式重命名。
- 不再创建空白分隔图，作品包里只放文案 TXT 和真实作品图。
- 成品库里凑满 14 个作品包时，会自动整理成 `作品集_001`、`作品集_002`，并生成同名 ZIP。

## 给别人安装

把这个目录发给对方后，在 PowerShell 里运行：

```powershell
.\install_work_package_tool.ps1 -TargetFolder "对方下载图片的文件夹" -LibraryName "成品库"
```

如果是团建工作流，可以这样装：

```powershell
.\install_work_package_tool.ps1 -TargetFolder "D:\Download\素材下载" -LibraryName "团建成品库"
```

安装后，对方平时只需要双击目标文件夹里的 `一键生成作品包.vbs`。

## 文件说明

- `一键生成作品包.vbs`：双击入口。
- `make_work_package.ps1`：核心处理脚本。
- `install_work_package_tool.ps1`：给别人安装到指定图片下载文件夹。
- `usage_zh.md`：用户使用说明，会在安装时复制到目标文件夹。
- `SKILL.md`：给 AI 助手维护这个工具时使用的规则说明。
